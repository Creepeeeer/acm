和普通莫队相似，只有离线区间查询，但此时不能通过普通双指针得出答案

**时间复杂度为$O(n\sqrt n)$**

**只增回滚莫队**

区间扩大的时候可以通过统计的信息得到答案，但是区间缩小的时候不能正确的得到答案

**做法**

排序规则还是按1.左边界所在块编号升序，2.右边界升序

对于l都在这个块的这些查询，当r<=br[i],暴力去得到答案,当处理r>br[i]，首先让ql=br[i]+1.qr=br[i],然后去处理每次查询。首先向右拓右边界，此时区间信息+答案都可以正确得到，此时记录一个答案last，然后向左拓左边界，此时区间信息+答案都可以正确得到。到了下一个查询，此时只动区间信息让ql=br[i]+1,不动答案因为此时区间减小并不能直接改变an，然后再让an=last（因为刚才得到last的区间就是现在这个）

[AT_joisc2014_c 歴史の研究 - 洛谷](https://www.luogu.com.cn/problem/AT_joisc2014_c)

    		auto&[l,r,id]=q[i];

    			qr++;
    			cnt[a[qr]]++;
    			re=max(re,1ll*nums[a[qr]]*cnt[a[qr]]);
    		}
    		last=re;
    		while(ql>l){
    			ql--;
    			cnt[a[ql]]++;
    			re=max(re,1ll*nums[a[ql]]*cnt[a[ql]]);
    		}

    		while(ql<br[bi[q[j].l]]+1){
    			cnt[a[ql]]--;
    			ql++;
    		}
    		re=last;
    		last=0;
    	}
    	for(int k=0;k<nums.size();k++)cnt[k]=0;
    }

**只删回滚莫队**

区间缩小的时候可以通过统计的信息得到答案，但是区间扩大的时候不能正确的得到答案

**做法：**

先按左边界的bi升序排序，然后再按右边界降序排序

对于一些左边界在同一个块的查询，一开始让左边界为bl[bi[q[i].l]，先移动右边界对齐，记录答案后再移动右边界得到答案，再把左边界回滚回去an=last

			re=last;
			while(ql>bd){
				ql--;
				cnt[a[ql]]++;
			}
			i++;
		}
